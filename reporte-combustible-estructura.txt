ESTRUCTURA DE DATOS PARA REPORTE DE CONSUMO DE COMBUSTIBLE
========================================================

Basado en el documento de la imagen "reporte_detallado.jpg"

TABLAS DE BASE DE DATOS NECESARIAS:
----------------------------------

1. TABLA: reporte_combustible
   - id (PRIMARY KEY)
   - codigo_reporte (VARCHAR) - Ej: "MA010171O"
   - fecha (DATE)
   - hora (TIME)
   - almacen (VARCHAR) - Ej: "SUB ALMACEN ALAMEDA RTL"
   - cliente_operacion (VARCHAR) - Ej: "F001-000015705"
   - referencia (VARCHAR)
   - valor_total (DECIMAL)
   - igv (DECIMAL)
   - total_soles (DECIMAL)
   - observaciones (TEXT)
   - created_at (TIMESTAMP)
   - updated_at (TIMESTAMP)

2. TABLA: empresa_datos
   - id (PRIMARY KEY)
   - razon_social (VARCHAR) - Ej: "ARIAS AYALA S.A.C"
   - ruc (VARCHAR) - Ej: "20439061"
   - usuario (VARCHAR) - Ej: "RAYALA"
   - sucursal (VARCHAR) - Ej: "PRINCIPAL"
   - guia_remision (VARCHAR) - Ej: "EG07-00008355"
   - direccion (TEXT) - Ej: "CAL LOS ANDES NRO. 155 URB. SAN GREGORIO, LIMA - LIMA - ATE"

3. TABLA: combustible_detalle
   - id (PRIMARY KEY)
   - reporte_id (FOREIGN KEY -> reporte_combustible.id)
   - codigo (VARCHAR) - Ej: "00002"
   - cantidad (DECIMAL) - Ej: "41.00", "66.50", "16.00", etc.
   - unidad_medida (VARCHAR) - Ej: "GALON"
   - descripcion (VARCHAR) - Ej: "DIESEL B5 S50", "DIESEL B5 S50 EN-003", etc.
   - codigo_producto (VARCHAR) - Ej: "EN-003", "EQ-002", "EN-004", "MC-003", "RE-005"
   - kilometraje (DECIMAL)
   - horometro (DECIMAL)
   - valor_unitario (DECIMAL)
   - importe_total (DECIMAL)
